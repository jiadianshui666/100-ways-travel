# 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
**百途旅行（100 Ways Travel）** — 探索 100 种旅行方式的内容平台。

### 1.2 产品定位
一个以「旅行方式」为核心组织维度的内容展示平台。与传统按目的地分类不同，百途旅行强调体验的类型和风格：美食之旅、极限冒险、城市探索、自然风光、文化体验……用户可以在平台上浏览、搜索、筛选感兴趣的旅行灵感。

### 1.3 目标用户
- **旅行爱好者**：寻找下一次旅行灵感
- **小众体验追求者**：厌倦标准化景点，寻求独特体验
- **内容创作者**：分享和发布旅行体验文章
- **平台运营者**：通过后台管理内容和分类

### 1.4 核心价值
- 按旅行方式而非目的地组织内容，降低选择疲劳
- 霓虹暗色视觉风格，打造沉浸式浏览体验
- 完整的前后台体系，支持内容运营闭环

## 2. 功能需求

### 2.1 前台用户端

#### 2.1.1 首页
- **全屏 Hero 区**：Canvas 动画渐变网格背景 + 浮动光球，品牌标语和行动号召
- **分类标签栏**：横向滚动，显示所有旅行分类（城市探索/自然风光/美食之旅/极限冒险/文化体验）及各分类体验数量，点击切换筛选
- **精选体验横滑区**：横向滚动展示 featured=true 的体验卡片，支持左右箭头导航
- **筛选栏**：视觉风格筛选（对应分类）、小众指数、排序（最新/价格升序/降序）
- **体验卡片网格**：响应式 3 列毛玻璃卡片，悬停动效，加载更多分页
- **状态全覆盖**：加载骨架屏、空状态提示、错误重试

#### 2.1.2 体验详情页
- **全宽视差 Hero 大图**：滚动驱动的图片视差效果
- **元信息条**：地点、价格、时长、作者，玻璃态卡片样式
- **信息卡片**：体验时长等级、预算区间、最佳季节（自动推算）
- **故事正文区**：分段渲染体验描述
- **图片画廊**：响应式网格布局，点击进入灯箱（键盘导航、缩略图指示器）
- **同分类推荐**：展示当前分类下的 3 个其他体验
- **SEO 优化**：`generateMetadata` 动态生成 OG 标签和描述

#### 2.1.3 搜索页
- **大输入框**：自动聚焦，带搜索图标和加载动画
- **400ms 防抖**：减少 API 请求频率
- **搜索结果网格**：复用体验卡片组件
- **空状态/加载态/错误态**：完整覆盖

#### 2.1.4 关于页
- 品牌故事 Hero 区
- 三列价值主张卡片（深度探索/灵感驱动/社区共创）
- 数据展示条
- 滚动渐入动效（IntersectionObserver）

### 2.2 后台管理端

#### 2.2.1 登录
- react-hook-form + zod 表单校验
- JWT 令牌签发与本地存储持久化
- 错误提示和加载状态

#### 2.2.2 仪表盘
- 统计卡片：总体验数、已发布数、精选数、浏览总量、分类数
- 快捷入口卡片

#### 2.2.3 体验管理
- 可排序表格（标题/价格/地点/日期）
- 客户端搜索过滤
- 创建/编辑表单：react-hook-form + zod 校验，自动 slug 生成，图片预览
- 删除确认弹窗 + Toast 反馈

#### 2.2.4 分类管理
- 列表展示（含体验数量）
- 内联表单创建/编辑
- 删除保护（有关联体验时无法删除）

## 3. 非功能需求

### 3.1 性能
- 首页 Largest Contentful Paint < 2.5s（服务端组件预渲染 + next/image 优化）
- 搜索防抖 400ms
- 体验卡片前 6 张设置 `priority` 预加载

### 3.2 可访问性
- 语义化 HTML5 标签（`<header>`, `<main>`, `<footer>`, `<nav>`, `<section>`）
- skip-to-content 链接
- aria-current / aria-expanded / aria-label / aria-modal
- 键盘导航（Escape 关闭弹窗，← → 切换灯箱）

### 3.3 兼容性
- 响应式设计：移动端 → 平板 → 桌面三档
- Chromium 浏览器（Playwright 测试目标）
- 现代浏览器（Chrome / Firefox / Edge / Safari）

### 3.4 安全性
- JWT HS256 签名，7 天过期
- bcryptjs 12 轮哈希
- Zod 输入校验（前后端双重验证）
- 管理端 API 统一鉴权中间件

## 4. 数据模型

3 张核心表：

- **User**：id, name, email, passwordHash, avatar, bio, role (USER/ADMIN)
- **Category**：id, name, slug, description, icon
- **TravelExperience**：id, title, slug, description, location, price, duration, images, featured, published, authorId (FK→User), categoryId (FK→Category)
